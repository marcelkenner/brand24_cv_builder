# CV Designer Brief

## Objective

Create a premium, modern-classic professional CV that visually matches the candidate’s headshot and presents them as polished, credible, ambitious, detail-oriented, and executive-ready. The final result must feel sharp, clean, contemporary, and formal, with zero unnecessary decoration.

This brief is intended to remove ambiguity. Follow it exactly unless the candidate later supplies role-specific changes.

---

## 1) Brand and Visual Direction

### Overall impression to communicate

The CV must communicate:

* professional maturity
* composure
* confidence without aggression
* precision and discipline
* strong executive presence
* modern professionalism
* high trustworthiness
* strong presentation standards

### Personality of the design

The design language should be:

* modern classic
* minimal
* structured
* restrained
* premium
* editorial rather than playful
* corporate rather than creative-agency
* sharp rather than soft
* confident rather than flashy

### Explicitly avoid

Do **not** use:

* bright saturated colors
* multiple accent colors
* infographic style skill charts
* progress bars
* timeline gimmicks
* overly decorative icons
* large icon sets for every line item
* playful or startup-casual visual cues
* asymmetrical experimental layouts
* exaggerated geometric elements
* textured backgrounds
* gradients
* shadows
* glassmorphism
* colored blocks behind large sections
* oversized initials or monograms
* script fonts
* condensed novelty fonts
* photo cutouts with creative shapes
* circular photo crops
* anything that competes with the headshot

---

## 2) Intended Role Positioning

The visual style should best fit these categories:

* consulting
* finance
* business development
* account management
* commercial leadership
* operations
* strategy
* legal-adjacent corporate roles
* senior corporate individual contributor roles
* manager-level roles

The design should still be flexible enough to work across general white-collar professional applications.

---

## 3) Document Format

### Primary version

Create the master CV in **A4 portrait**.

### Secondary export

Also create a **US Letter portrait** adaptation with line breaks adjusted manually so it does not look auto-scaled.

### Length

* Target: 1 page if experience is concise and high-impact.
* Allow 2 pages if experience justifies it.
* Never compress type or spacing just to force 1 page.
* If 2 pages are used, page 2 must feel intentionally designed, not like overflow.

### Output files required

Deliver:

* editable source file in Figma, InDesign, or Word-compatible layout file
* press-clean PDF for sharing
* ATS-friendly Word version with the same hierarchy but simplified formatting

---

## 4) Layout Architecture

### Recommended layout

Preferred layout: **single-column primary reading flow**.

Reason:

* It looks more executive and serious.
* It preserves readability.
* It is more ATS-friendly.
* It aligns better with the headshot’s formal tone.

### Optional variation

A restrained two-column layout is acceptable only if done very carefully:

* left column narrow: contact / skills / languages / certifications
* right column wide: profile / experience / education / achievements

If using two columns:

* left column width: 28% to 32%
* right column width: 68% to 72%
* gutter: 6 mm to 8 mm
* no dark sidebar block
* sidebar background must remain white or a barely tinted neutral only if necessary

### Page margins

For A4:

* top: 14 mm to 18 mm
* bottom: 14 mm to 18 mm
* left: 16 mm to 20 mm
* right: 16 mm to 20 mm

Recommended exact setting:

* top 16 mm
* bottom 16 mm
* left 18 mm
* right 18 mm

### Grid

Use a strict baseline and spacing system.

Recommended spacing scale:

* 4 pt micro spacing
* 8 pt standard spacing
* 12 pt medium spacing
* 16 pt section spacing
* 24 pt major separation

Nothing should be spaced arbitrarily.

---

## 5) Headshot Treatment

### Role of the headshot

The headshot should support the brand impression, not dominate the page.
It must look intentional, discreet, premium, and aligned with formal business presentation.

### Use of photo

* Include photo only for markets/industries where photos on CVs are standard or acceptable.
* For ATS or regions where photos are discouraged, create a no-photo version as well.

### Placement

If photo is included, preferred placement order:

1. top right in the header area
2. top left only if aligned with a strong editorial header

Best recommendation for this specific visual direction:

* top right placement
* aligned with name block on same vertical band

### Crop

* rectangular portrait crop
* vertical orientation
* head and upper shoulders visible
* no circular crop
* no rounded corners beyond a subtle 2–4 px radius if used at all
* avoid tight crop that cuts into hair shape
* preserve the clean background blur where possible

### Photo size

Recommended print/display size on page:

* width: 28 mm to 35 mm
* height proportional to original aspect ratio

Safe default:

* 32 mm wide

### Photo styling

* no filters
* no background removal
* no drop shadow
* no colored frame
* no heavy border
* at most a hairline neutral border if needed for separation

---

## 6) Color System

### Core palette intent

Use a restrained corporate palette with one accent color only.
The palette must match the sophistication of the headshot.

### Primary recommendation

Use:

* charcoal for primary text
* soft warm white background
* deep burgundy accent

This works especially well because it quietly harmonizes with the tie color.

### Exact color palette

Primary text:

* #1F2328 or #222222

Secondary text:

* #4A4F57 or #555555

Muted text / metadata:

* #6B7280 or #70757D

Rules / dividers:

* #D9DDE3 or #E1E4E8

Background:

* #FFFFFF

Accent option A (preferred burgundy):

* #8A1538
* alternate slightly softer burgundy: #7A1F3D

Accent option B (safe navy alternative):

* #1F3A5F

### Accent usage rules

Accent color may be used only for:

* section headings
* subtle horizontal dividers
* name underline or micro rule
* link highlights
* selected labels

Accent color must **not** be used for:

* large background fills
* body text
* icons everywhere
* decorative blocks
* heavy borders

### Color ratio

Approximate distribution:

* 85–90% neutrals
* 10–15% accent

---

## 7) Typography

### Typographic personality

The typography should feel:

* sharp
* modern
* highly legible
* professional
* premium without looking luxury-branded

### Recommended fonts

Use one of these combinations:

**Option 1 – safest and strongest**

* Headings / name: Inter SemiBold or Bold
* Body: Inter Regular / Medium

**Option 2 – classic corporate**

* Headings / name: Aptos Display or Calibri Bold
* Body: Aptos or Calibri

**Option 3 – slightly editorial but safe**

* Headings: Source Sans 3 SemiBold
* Body: Source Sans 3 Regular

### Font rules

* Use a maximum of 2 font families.
* Prefer 1 family with multiple weights.
* No serif unless using a very restrained editorial version and the rest remains modern.
* No script, no display novelty fonts.

### Font sizes

Recommended A4 hierarchy:

* Name: 22–28 pt
* Job title / positioning line: 10.5–12 pt
* Section headings: 10–11.5 pt
* Company / role headings: 10.5–11.5 pt
* Body text: 9.5–10.5 pt
* Metadata text: 8.5–9 pt
* Footer/page number if used: 8 pt

Safe default hierarchy:

* Name: 26 pt
* Positioning line: 11 pt
* Section headings: 10.5 pt
* Company/role line: 10.5 pt
* Body: 10 pt
* Metadata: 8.75 pt

### Line spacing

* Body leading: 1.3 to 1.45
* Dense achievement bullets may go slightly tighter, but never cramped
* Section heading tracking can be slightly increased for polish

### Text emphasis rules

Use bold only for:

* candidate name
* company names or role titles, but choose one system and keep consistent
* essential metrics if used sparingly

Use italics minimally:

* only for publications or optional context lines if necessary

Do not overuse bold. The page should not look noisy.

---

## 8) Header Design

### Header contents

The header must contain:

* full name
* current target title or professional label
* city and country or city only
* phone
* email
* LinkedIn
* portfolio or website if relevant
* photo, if the photo version is used

### Header hierarchy

1. Full name
2. Professional title / positioning line
3. Contact line
4. Photo aligned on the same band or same top region

### Name styling

* large but not oversized
* either all title case or small caps-inspired uppercase styling if cleanly executed
* no decorative underline except possibly a thin accent rule below the header

### Professional title line examples

This line should be concise and targeted, for example:

* Business Development Manager
* Strategy & Operations Professional
* Commercial Manager
* Account Executive | Enterprise Sales
* Corporate Finance Analyst

### Contact line styling

* single line if space allows
* separated by bullet dots, vertical dividers, or spacing
* all contact items should be clickable in digital PDF
* icon use optional and minimal

### Header spacing

* generous white space below the header before first section
* the header must not feel congested even if photo is included

---

## 9) Section Order

Recommended section order for a professional corporate CV:

1. Header
2. Professional Summary
3. Core Competencies / Areas of Expertise
4. Professional Experience
5. Education
6. Certifications or Training
7. Languages
8. Technical Skills
9. Additional Information / Awards / Interests only if relevant

### Section inclusion rules

* Do not include every possible section by default.
* Only include sections that add hiring value.
* Hobbies/interests should only appear if they support the professional narrative or help humanize without reducing seriousness.

---

## 10) Professional Summary

### Purpose

A 3–5 line summary that positions the candidate clearly and strategically.

### Tone

* direct
* polished
* commercially aware
* outcomes-focused
* not inflated
* no clichés

### Must avoid

* “hardworking team player” language
* generic self-description
* first person pronouns
* vague buzzword stacking

### Structure

Sentence 1:

* who the candidate is professionally

Sentence 2:

* years or level of experience + sector/function

Sentence 3:

* strongest commercial/operational/strategic strengths

Sentence 4:

* measurable value or target direction

### Length

* 45 to 80 words ideal
* maximum around 100 words

---

## 11) Core Competencies / Expertise Section

### Visual treatment

Show as either:

* clean keyword rows separated by dots or dividers
* a 2-column text list
* a compact horizontal wrap list

### Avoid

* skill bars
* star ratings
* progress meters
* generic icon tags

### Example categories

* Client Relationship Management
* Commercial Strategy
* Business Development
* Stakeholder Management
* Contract Negotiation
* Sales Pipeline Management
* Market Analysis
* Process Improvement
* Cross-functional Collaboration
* Financial Reporting

### Quantity

* 8 to 14 competencies maximum
* only high-value, role-relevant items

---

## 12) Professional Experience

### Order

Reverse chronological.

### Entry structure

Each role should include:

* job title
* company name
* location
* employment dates (month year – month year)
* optional one-line company description only if the company is not well known and context helps
* 3 to 6 bullets for recent roles
* fewer bullets for older roles

### Visual hierarchy choice

Choose one consistent pattern:

* role title bold, company regular
  or
* company bold, role title regular

Recommended for this CV:

* **Role title bold**
* **Company name medium or regular small caps-style emphasis**

### Date placement

Options:

* right-aligned on same line as role/company
* beneath role/company in lighter metadata styling

Best recommendation:

* right-aligned dates for clean executive rhythm

### Bullet styling

* use simple round bullets or short en-dash bullets
* no icons
* no paragraph blocks masquerading as bullets

### Bullet writing rules

Every bullet should preferably demonstrate one or more of:

* achievement
* ownership
* scale
* improvement
* money impact
* efficiency gain
* commercial result
* stakeholder scope
* client impact

### Bullet formula

Recommended formula:
**Action + scope/context + measurable result**

Example structure:

* Led [initiative/process/account portfolio] across [team/client/region], resulting in [quantified outcome].

### Bullet length

* 1 line ideal
* 2 lines acceptable
* avoid 3+ lines unless essential

### Quantification rules

Where possible include:

* revenue
* growth percentage
* size of accounts managed
* number of stakeholders
* cost savings
* process time reduction
* conversion improvement
* budget size
* project value
* regional scope

### Tense rules

* present tense for current role
* past tense for previous roles

### Spacing

* tight but breathable space between bullets
* slightly larger gap between role entries

---

## 13) Education

### Include

* degree
* institution
* location
* graduation year or attendance years
* classification/GPA only if strong and relevant
* thesis or relevant modules only if early-career and important

### Order

Newest/highest qualification first.

### Optional additions

If strong and relevant:

* academic honors
* relevant coursework
* scholarships
* society leadership

### Visual treatment

Keep it clean and shorter than experience unless candidate is recent graduate.

## 14) Certifications / Training

* renewal status if important

## 17) Optional Additional Sections

Only include these if they clearly strengthen the CV:

## 18) ATS Compatibility Rules

Even the designed version should remain ATS-conscious.

### Must do

* use real text, not outlined text
* avoid placing core content inside text boxes if exporting to Word
* keep section headings conventional
* use simple bullet characters
* ensure logical reading order
* ensure links remain clickable
* avoid multi-column complexity in ATS export

### Must not do

* no tables for main content in ATS version
* no charts
* no icons replacing labels
* no image-only text
* no headers/footers containing crucial information in ATS version

### Required deliverable

Create:

1. designed PDF version
2. simplified ATS Word version

Both must contain the same underlying content.

---

## 19) White Space and Density Control

### Desired density

The page should feel efficient and premium, not sparse and not cramped.

### Rules

* no orphan section headings at bottom of page
* no single bullet stranded on next page if avoidable
* no dense walls of text
* no giant empty gaps caused by photo placement
* align all left edges consistently
* baseline rhythm must feel controlled

### Visual test

At a 50% zoom, the document should look:

* balanced
* clean
* disciplined
* easy to scan in 6–8 seconds

---

## 20) Divider and Line Rules

### Horizontal rules

Allowed, but subtle.
Use only:

* thin line 0.5 pt to 1 pt
* neutral grey or accent color at low visual dominance

### Use cases

* beneath header
* between major sections if needed

### Avoid

* thick rules
* boxed section containers
* too many dividing lines

---

## 21) Iconography Rules

* mixed icon styles

## 26) Final Quality-Control Checklist for Designer

Before delivery, confirm all of the following:

* name is the strongest visual element
* photo does not overpower the page
* hierarchy is instantly scannable
* section headings are consistent in size, weight, color, and spacing
* all dates use one format only
* punctuation style is consistent throughout
* bullet alignment is consistent
* line breaks are intentional, not accidental
* accent color use is restrained and repeated consistently
* margins are even
* spacing follows a system
* no widows/orphans where possible
* PDF links are clickable
* ATS version contains no layout traps
* spelling, capitalization, and tense are consistent
* the CV feels premium, modern, formal, and credible
* the design visually matches the seriousness and polish of the headshot

---

## 27) One-Sentence Creative Direction

Design a **modern-classic executive CV** with a clean white base, charcoal typography, a restrained burgundy accent, disciplined spacing, subtle hierarchy, and optional top-right rectangular headshot placement, so the overall impression is polished, confident, credible, and corporate.

## 31) Homepage Product Positioning And Case-Study Structure

The homepage must show two things at once:

* a useful end product for recruiters
* clear proof that the candidate can design, govern, and implement AI-enabled workflows end to end

### Product framing

Position the project as a small internal-style AI product, not as a generic portfolio page.

Recommended project title:

* **CV Builder — AI-Assisted Resume Tailoring System**

Recommended framing statement:

* “I had a real workflow problem: adapting my CV for different roles without losing factual accuracy or formatting quality. I designed an AI-assisted system that turns one verified experience base into multiple recruiter-ready CV variants and explainable outputs.”

### Homepage goals

The homepage must satisfy two reading speeds:

* a recruiter should understand the value and download a relevant CV within 30 seconds
* a hiring manager should be able to inspect the AI workflow, rules, architecture, and delivery choices within 5 to 10 minutes

### Required homepage modes

The page should present two clearly separated but connected modes:

#### Mode A: Recruiter View

Must include:

* clean landing section
* short professional introduction
* visible list of available CV variants as cards
* per-card metadata: target role, language, short summary
* per-card actions: `Download PDF`
* optional per-card actions: `View online`, `Listen`

#### Mode B: Project Case Study

Must include:

* problem statement
* success criteria
* why AI was useful
* technology selection
* system design
* prompt and system rules
* validation and guardrails
* examples of outputs
* lessons learned
* next improvements

### Required information architecture

The homepage should follow this structure:

1. Hero
2. CV workspace
3. What this demonstrates
4. Why I built this
5. How I built it
6. What we fixed

### Section intent

#### Hero

The first screen must make the immediate recruiter action obvious, while also
introducing the narrative sections without repeating them in a second summary
card.

Required behavior:

* primary recruiter download CTA
* direct CTA into the CV workspace
* lighter jump links into `Why I built this`, `How I built it`, and `What we fixed`
* no duplicate `What this demonstrates` summary panel in the hero
* if icons are used, keep them to restrained utility affordances and structured labels such as CTAs, metadata rows, document links, and clearly named problem/fix/result markers

#### CV workspace

This section should feel like a document workspace, not a product-card grid.

Required behavior:

* left-side variant selector
* large live preview on the right
* sticky actions for `View online`, `Listen`, and `Download PDF`
* quick summary of how the selected variant differs from the others

#### Why I built this

This section should explain why the project started, why the build approach was chosen, and which documents turned the first idea into an executable plan.

Recommended coverage:

* seeing the AI Adoption Manager post and deciding to apply
* choosing to build something more deliberate than sending a standard PDF
* selecting Codex as the implementation partner because it was already part of the working toolchain
* researching the HTML/CSS-first plus Playwright route to preserve browser-to-PDF fidelity
* listing the key requirement and implementation docs that defined the first pass
* acknowledging the refinement loop after the first pass to remove mistakes and drift

#### How I built it

Show one explicit workflow from verified profile data to recruiter-ready PDF output.

#### What we fixed

This section should explain the most important implementation problems discovered during the build and how they were corrected.

Recommended coverage:

* random or awkward page breaks in early print/PDF output
* the first section-level pagination fix
* the blank-space regression caused by moving whole sections
* the refinement to subsection-aware units such as separate experience roles
* the final measured print-route pagination pass before export

### Final positioning rule

The homepage must communicate:

* a polished product that is useful immediately
* a mini case study that proves workflow design and implementation judgment

Do not present this as an oversized platform. The correct signal is a small, polished product plus a clearly explained case study.
