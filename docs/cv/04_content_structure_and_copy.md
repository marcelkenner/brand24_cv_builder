# Content Structure And Copy

This document expands `docs/project_description.md` sections 9 through 17, 22 through 24, and 29.

## Section Order

Default section order:

1. Header
2. Professional Summary
3. Core Competencies
4. Professional Experience
5. Education
6. Certifications or Training
7. Languages
8. Technical Skills
9. Additional information only when it adds hiring value

Do not add sections just to fill space.

## Professional Summary

Purpose:

- position the candidate quickly and strategically

Rules:

- 3 to 5 lines
- ideally 45 to 80 words
- direct, polished, commercially aware, and measurable
- no clichés, no first-person voice, no vague buzzword stacks

Recommended sentence pattern:

1. who the candidate is professionally
2. years or level of experience and sector
3. strongest functional strengths
4. measurable value or target direction

## Core Competencies

Allowed presentation:

- compact keyword rows
- restrained two-column text list
- horizontal wrapped list

Do not use:

- bars
- stars
- meters
- icon tags

Target count:

- 8 to 14 items

Only include high-value, role-relevant competencies.

## Professional Experience

Order:

- reverse chronological

Each role should include:

- job title
- company name
- location
- employment dates
- optional one-line company description when needed for context
- 3 to 6 bullets for recent roles
- fewer bullets for older roles

Hierarchy:

- recommended pattern is role title emphasized, company secondary
- dates should be right-aligned when possible for cleaner rhythm

Bullet rules:

- use simple bullets or short dashes
- each bullet should show ownership, achievement, scope, impact, or measurable improvement
- preferred formula is action plus context plus result
- 1 line is ideal, 2 lines acceptable, avoid 3 or more lines
- use present tense for the current role and past tense for prior roles

Quantify whenever possible:

- revenue
- growth
- account size
- stakeholder count
- savings
- time reduction
- conversion improvement
- budget
- project value
- regional scope

## Education, Certifications, Languages, And Skills

Education should stay concise unless the candidate is early-career.

Include:

- degree or qualification
- institution
- location
- year or attendance range
- honors or GPA only when strong and relevant

Certifications should contain:

- certification name
- issuing body
- year
- renewal status when it matters

Languages should be plain text, for example:

- English — Native
- German — Professional Working Proficiency

Technical skills should be:

- role-relevant
- grouped
- text-based

Avoid visual rating systems in all of these sections.

## Optional Sections

Only include optional sections if they strengthen the hiring case:

- awards
- selected achievements
- publications
- board or volunteer leadership
- military service
- memberships
- security clearance

If a section exists only to occupy space, remove it.

## Tone Of Copy

The writing should sound:

- credible
- concise
- commercially literate
- measurable
- senior enough for responsibility
- confident without arrogance

Avoid phrases like:

- results-driven professional
- team player
- go-getter
- hard-working
- think outside the box
- self-starter
- dynamic professional

Prefer concrete verbs:

- managed
- led
- negotiated
- delivered
- increased
- reduced
- built
- developed
- improved
- secured
- expanded
- streamlined

## Candidate Content Intake Checklist

Collect all of the following before finalizing content:

- full legal or professional name
- preferred display name if different
- target title
- city and country
- phone
- email
- LinkedIn URL
- website or portfolio if any
- total years of experience
- industry background
- current role
- target role
- top strengths
- top measurable wins
- sectors worked in

For each job, collect:

- title
- company
- location
- start and end dates
- company context if not obvious
- team size
- seniority context
- responsibilities
- achievements
- quantified results
- tools used when relevant

Also collect:

- education details
- hard skills
- software and tools
- languages
- certifications
- licenses
- optional awards or supporting details

## Prioritization Rules

When space is tight, keep content in this order:

1. current and recent experience
2. quantified achievements
3. professional summary
4. core competencies
5. education
6. certifications
7. languages
8. technical tools
9. additional interests

Remove weak content before shrinking type or tightening layout.

## Copywriter Rules

If content is being rewritten:

- convert duties into outcomes
- quantify claims where possible
- remove filler
- keep bullets syntactically parallel
- tailor the title and summary to the target role
- shorten older experience
- keep tense correct
- remove statements that do not strengthen the hiring case

## Implementation Notes

For the app’s sample CV content:

- use concise, high-signal placeholder copy that follows these rules
- avoid dummy text that breaks the tone of the system
- structure the data model so sections can be omitted cleanly when they add no value
